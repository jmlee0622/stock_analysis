# 실시간 주식/암호화폐 데이터 분석 프로젝트

> 실시간 암호화폐 데이터를 수집하고, AI로 뉴스를 분석해 가격 변동과의 연관성을 파악하는 시스템

---

## 전체 아키텍처

```
[Finnhub WebSocket]  →  실시간 거래 데이터 (BTC, ETH)
        ↓
[Spring Boot]  →  Kafka Producer  →  Kafka 브로커  →  Kafka Consumer  →  TimescaleDB

[Finnhub REST API]  →  뉴스 (1시간마다)
        ↓
[Spring Boot]  →  Claude AI 감성 분석  →  TimescaleDB

[React Frontend (Vite)]  →  axios  →  Spring Boot REST API  →  Redis 캐시  →  TimescaleDB / ChromaDB
```

## Step 1. 실시간 데이터 파이프라인

### 목표
Finnhub WebSocket → Kafka → TimescaleDB로 실시간 거래 데이터 수집

### 핵심 개념

**WebSocket** : HTTP와 달리 한 번 연결하면 서버가 데이터를 계속 밀어줌. Finnhub이 1초에 수십 건 거래 데이터를 전송.

**Kafka** : 데이터 유실 방지 + 여러 서비스가 동시에 소비 가능한 메시지 큐. Finnhub에서 직접 DB에 저장해도 되지만 Kafka를 두면 처리 속도 차이를 완충.

```java
// FinnhubWebSocketClient.java
public void onOpen(ServerHandshake handshake) {
    send("{\"type\":\"subscribe\",\"symbol\":\"BINANCE:BTCUSDT\"}");
    send("{\"type\":\"subscribe\",\"symbol\":\"BINANCE:ETHUSDT\"}");
}

public void onMessage(String message) {
    stockProducer.send(message); // Kafka로 전달
}
```

## Step 2. Kafka Consumer + TimescaleDB 저장

### 목표
Kafka 메시지를 파싱해 TimescaleDB에 저장

### Finnhub 데이터 형태
```json
{"type":"trade","data":[{"s":"BINANCE:BTCUSDT","p":7296.89,"v":0.011,"t":1575526691134}]}
```
`s`=symbol, `p`=price, `v`=volume, `t`=timestamp(ms)

```java
// StockConsumer.java
@KafkaListener(topics = "stock-data", groupId = "stock-group")
public void consume(String message) {
    FinnhubTradeResponse response = objectMapper.readValue(message, FinnhubTradeResponse.class);
    if (!"trade".equals(response.getType())) return;

    List<StockTrade> trades = response.getData().stream()
        .map(d -> StockTrade.builder()
            .symbol(d.getSymbol()).price(d.getPrice())
            .tradeTime(Instant.ofEpochMilli(d.getTimestamp()))
            .build())
        .toList();

    stockTradeRepository.saveAll(trades); // 배치 INSERT
}
```

## Step 4-②. REST API + Redis 캐시

### 목표
조회 API 제공, Redis로 DB 부하 감소

### 핵심 개념

**캐시 동작** : 메서드 호출 → Redis에 키 있으면 즉시 반환(캐시 히트) → 없으면 DB 조회 후 Redis 저장(캐시 미스)

**TTL 설정 기준**
| 캐시 | TTL | 이유 |
|------|-----|------|
| latestTrade, latestTrades | 5초 | 실시간 시세라 자주 바뀜 |
| newsList, newsBySentiment | 1시간 | 스케줄러 갱신 주기와 동일 |

```java
// TradeService.java
@Cacheable(value = "latestTrade", key = "#symbol")
public TradeResponse getLatestBySymbol(String symbol) { ... }

@Cacheable(value = "latestTrades")
public List<TradeResponse> getLatestAll() { ... }
```

**API 엔드포인트**
```
GET /api/trades/latest              → BTC + ETH 최신 시세
GET /api/trades/latest?symbol=...   → 특정 심볼 시세
GET /api/news                       → 최신 뉴스 20건
GET /api/news?sentiment=POSITIVE    → 감성별 필터링
```

## Step 4-③. JWT 보안

### 목표
로그인한 사용자만 API 접근 가능

### 핵심 개념

**세션 vs JWT**
```
세션: 서버가 로그인 상태 저장 → 요청마다 세션 조회
JWT:  서버가 토큰 발급(저장 X) → 요청마다 서명 검증만
```

```java
// SecurityConfig.java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/auth/**", "/actuator/**").permitAll()
    .anyRequest().authenticated()
)
.addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)
```

## Step 5. ChromaDB 벡터 검색

### 목표
뉴스를 벡터로 변환해 의미 기반 유사도 검색 제공

### 핵심 개념

**벡터 DB** : 텍스트를 숫자 배열(벡터)로 변환해 저장. 키워드가 없어도 "의미적으로 비슷한" 문서 검색 가능.

**임베딩 모델 (AllMiniLM-L6-v2)** : 텍스트 → 384차원 벡터로 변환. ONNX 형식으로 로컬 실행(무료).

```java
// NewsVectorService.java
// 저장: 텍스트 → 벡터 → ChromaDB
public void storeNewsEmbedding(Long newsId, String headline, String summary, String sentiment) {
    Embedding embedding = embeddingModel.embed(headline + " " + summary).content();
    embeddingStore.add(embedding, TextSegment.from(text, metadata));
}

// 검색: 검색어 → 벡터 → 유사도 검색
public List<EmbeddingMatch<TextSegment>> searchSimilarNews(String query, int maxResults) {
    Embedding queryEmbedding = embeddingModel.embed(query).content();
    return embeddingStore.findRelevant(queryEmbedding, maxResults);
}
```
