# 실시간 주식/암호화폐 데이터 분석 프로젝트

> 실시간 암호화폐 데이터를 수집하고, AI로 뉴스를 분석해 가격 변동과의 연관성을 파악하는 시스템

---

## 전체 아키텍처

```
[Finnhub WebSocket]
        ↓ 실시간 거래 데이터 (BTC, ETH)
[Spring Boot - FinnhubWebSocketClient]
        ↓
[Kafka Producer] → [Kafka 브로커] → [Kafka Consumer]
                                            ↓
                                   [TimescaleDB - stock_trades 테이블]


[Finnhub REST API]
        ↓ 암호화폐/주식 뉴스 (1시간마다, 자동 스케줄러)
[Spring Boot - NewsService]
        ↓
[FinBERT - HuggingFace 감성 분석]  ← router.huggingface.co/hf-inference/models/ProsusAI/finbert
        ↓ POSITIVE / NEGATIVE / NEUTRAL
[TimescaleDB - news_articles 테이블]
        ↓ 뉴스 수집 완료 시 @CacheEvict로 Redis 캐시 초기화
```

## Step 1. 실시간 데이터 파이프라인 구축

### 목표
Finnhub에서 BTC/ETH 실시간 거래 데이터를 받아 Kafka로 전달

### 핵심 개념

**WebSocket이란?**
- 일반 HTTP는 클라이언트가 요청해야만 서버가 응답
- WebSocket은 한 번 연결하면 서버가 데이터를 실시간으로 계속 밀어줌
- Finnhub이 WebSocket으로 1초에 수십 건의 거래 데이터를 전송

**Kafka란?**
- 메시지 큐(중간 저장소) 역할
- Finnhub → 직접 DB 저장도 가능하지만, Kafka를 중간에 두면:
  - 데이터 유실 방지
  - 여러 서비스가 같은 데이터를 동시에 소비 가능
  - 처리 속도 차이를 완충

// Java-WebSocket 라이브러리로 Finnhub에 연결

        send("{\"type\":\"subscribe\",\"symbol\":\"BINANCE:ETHUSDT\"}");
    }

    @Override
    public void onMessage(String message) {
        stockProducer.send(message); // Kafka로 전달
    }
}

## Step 2. Kafka Consumer + TimescaleDB 저장

### 목표
Kafka에서 데이터를 받아 파싱 후 DB에 저장

### Finnhub이 보내는 데이터 형태
```json
{
  "type": "trade",
  "data": [
    {"s": "BINANCE:BTCUSDT", "p": 7296.89, "v": 0.011, "t": 1575526691134},
    {"s": "BINANCE:ETHUSDT", "p": 150.23,  "v": 0.5,   "t": 1575526691135}
  ]
}
```
- `s` → symbol (종목명)
- `p` → price (가격)
- `v` → volume (거래량)
- `t` → timestamp (밀리초 단위 시간)

> 메시지 1개 안에 거래 데이터가 배열로 여러 건 들어올 수 있음

### FinnhubTradeResponse.java (DTO)
```java
// JSON → Java 객체 변환을 위한 틀
// @JsonProperty로 JSON 키("s")와 Java 필드명(symbol)을 연결
@JsonIgnoreProperties(ignoreUnknown = true) // 모르는 필드는 무시
public class FinnhubTradeResponse {

    @JsonProperty("type")
    private String type;

    @JsonProperty("data")
    private List<TradeData> data;

    public static class TradeData {
        @JsonProperty("s") private String symbol;
        @JsonProperty("p") private Double price;
        @JsonProperty("v") private Double volume;
        @JsonProperty("t") private Long timestamp;
    }
}
```

### StockTrade.java (Entity)
```java
// DB 테이블과 1:1 매핑되는 클래스
// @Entity 붙이면 JPA가 자동으로 테이블 생성
@Entity
@Table(name = "stock_trades")
public class StockTrade {
    @Id @GeneratedValue
    private Long id;
    private String symbol;
    private Double price;
    private Double volume;
    private Instant tradeTime;
}
```

### StockConsumer.java
```java
@KafkaListener(topics = "stock-data", groupId = "stock-group")
public void consume(String message) {
    // 1. JSON 문자열 → FinnhubTradeResponse 객체로 변환
    FinnhubTradeResponse response = objectMapper.readValue(message, FinnhubTradeResponse.class);

    // 2. "trade" 타입만 처리 (ping 등 다른 타입 무시)
    if (!"trade".equals(response.getType())) return;

    // 3. data 배열을 StockTrade 엔티티 리스트로 변환
    List<StockTrade> trades = response.getData().stream()
        .map(data -> StockTrade.builder()
            .symbol(data.getSymbol())
            .price(data.getPrice())
            .volume(data.getVolume())
            .tradeTime(Instant.ofEpochMilli(data.getTimestamp())) // ms → Instant
            .build())
        .toList();

    // 4. 한 번에 DB 저장 (saveAll = 배치 INSERT)
    stockTradeRepository.saveAll(trades);
}
```

### 데이터 흐름 상세
```
Kafka 메시지(JSON 문자열)
  → objectMapper.readValue()  : 문자열 → Java 객체
  → type 필터링               : "trade"만 통과
  → stream().map()            : 배열 → 엔티티 리스트 변환
  → saveAll()                 : 한 번에 DB INSERT
```

## Step 3. AI 뉴스 감성 분석

### 목표
Finnhub 뉴스 API로 암호화폐 뉴스를 가져와 Claude AI로 감성 분석 후 DB 저장

### 핵심 개념

**LangChain4j란?**
- Claude, GPT 등 AI 모델을 Java에서 쉽게 쓸 수 있게 해주는 라이브러리
- 파이썬의 LangChain을 Java로 구현한 버전

**감성 분석이란?**
- 텍스트가 긍정적인지(POSITIVE) 부정적인지(NEGATIVE) 중립인지(NEUTRAL) 판단
- 예: "비트코인 ETF 승인" → POSITIVE / "SEC 규제 강화" → NEGATIVE

### NewsService.java (전체 흐름)
```java
public void fetchAndAnalyzeNews() {
    // 1. Finnhub REST API로 최신 뉴스 가져오기
    FinnhubNewsDto[] newsDtos = restTemplate.getForObject(url, FinnhubNewsDto[].class);

    for (FinnhubNewsDto dto : newsDtos) {
        // 2. 이미 분석한 뉴스는 건너뜀
        if (newsArticleRepository.existsByNewsId(dto.getId())) continue;

        // 3. Claude로 감성 분석
        SentimentResult result = sentimentAnalysisService.analyze(
            dto.getHeadline(), dto.getSummary()
        );

        // 4. 결과 DB 저장
        newsArticleRepository.save(NewsArticle.builder()
            .newsId(dto.getId())
            .headline(dto.getHeadline())
            .sentiment(result.getSentiment())
            .sentimentReason(result.getReason())
            .analyzedAt(Instant.now())
            .build());
    }
}
```

## Step 4-②. REST API + Redis 캐시

### 목표
조회 API를 만들고 Redis로 캐시를 적용해 DB 부하 감소

### 핵심 개념

**Redis 캐시란?**
- 자주 쓰는 데이터를 메모리(RAM)에 저장해두는 고속 저장소
- TimescaleDB(디스크, ~수십ms) vs Redis(메모리, ~1ms 미만)
- 캐시 히트: Redis에 데이터 있으면 DB 조회 없이 즉시 반환
- 캐시 미스: Redis에 없으면 DB 조회 → Redis 저장 → 반환

**@Cacheable 동작 방식**
```
메서드 호출
  → Redis에 해당 키가 있냐? (데이터 비교 X, 키 존재 여부만 확인)
  → 있으면: DB 조회 안 하고 Redis에서 바로 반환 (캐시 히트)
  → 없으면: DB 조회 → Redis에 저장 → 반환 (캐시 미스)
  → TTL 만료 후: 키 자동 삭제 → 다음 요청 시 DB 재조회
```

**캐시 키 구조**
```
key 없음: "latestTrades"                      → 키 하나
key 있음: "latestTrade::BINANCE:BTCUSDT"      → symbol마다 키 생성
          "latestTrade::BINANCE:ETHUSDT"       (value::key 형태)
```

**TTL 설정 기준**
| 캐시 | TTL | 이유 |
|------|-----|------|
| latestTrade, latestTrades | 5초 | 실시간 시세라 자주 바뀜 |
| newsList, newsBySentiment | 1시간 | 스케줄러 갱신 주기와 동일 |

**@Cacheable 주의사항**
- 빈 결과(`[]`)도 캐싱됨 → 데이터가 추가돼도 TTL 만료 전까지 반영 안 됨
- 데이터를 추가하는 메서드에는 반드시 `@CacheEvict`로 캐시를 비워줘야 함
```java
// 뉴스 수집 완료 시 관련 캐시 전체 초기화
@CacheEvict(value = {"newsList", "newsByCategory", "newsBySentiment", "newsBySentimentAndCategory"}, allEntries = true)
public void fetchAndAnalyzeNews(...) { ... }
```

**레이어드 아키텍처**
```
Controller  → 요청 받고 응답 내보내기
Service     → 비즈니스 로직 (@Cacheable 적용)
Repository  → DB 조회
```
Controller가 DB를 직접 건드리지 않고 Service를 거치는 구조

**Entity vs DTO**
- Entity: DB 테이블과 1:1 매핑 (내부용)
- DTO: API 응답 전용 클래스 (외부 노출용)
- Redis 직렬화를 위해 DTO에 Serializable 구현 필수

## Step 4-③. JWT 보안

### 목표
토큰 기반 인증으로 API 보호 - 로그인한 사용자만 시세/뉴스 API 접근 가능

### 핵심 개념

**JWT (JSON Web Token)란?**
- 서버가 로그인 상태를 저장하지 않고 토큰으로 인증하는 방식
- 토큰 구조: `Header.Payload.Signature`
  - Header: 알고리즘 정보
  - Payload: username, 발급시간, 만료시간
  - Signature: 비밀키로 서명 (위변조 감지)

**세션 vs JWT**
```
세션: 로그인 → 서버가 세션 저장 → 요청마다 세션 조회
JWT:  로그인 → 서버가 토큰 발급(저장 X) → 요청마다 서명 검증
```

**인증 흐름**
```
POST /auth/login → JWT 발급 (유효기간 24시간)
        ↓
GET /api/trades/latest
Header: Authorization: Bearer eyJhbGci...
        ↓
JwtFilter → 토큰 검증 → 통과 or 403
```

### 공개/보호 경로
```
공개 (토큰 불필요): /auth/register, /auth/login, /actuator/**
보호 (토큰 필요):  /api/trades/**, /api/news/**
```

## Step 5. ChromaDB 벡터 검색

### 목표
뉴스를 벡터로 변환해 ChromaDB에 저장하고, 의미 기반 유사도 검색 제공

### 핵심 개념

**벡터 DB란?**
- 텍스트를 숫자 배열(벡터)로 변환해 저장
- "의미적으로 비슷한" 문서를 검색 가능
- 키워드가 없어도 관련 뉴스 검색 가능

**임베딩 모델 (AllMiniLM-L6-v2)**
- 텍스트 → 384차원 벡터로 변환
- ONNX 형식으로 내장 (로컬 실행, 무료)
- `langchain4j-embeddings-all-minilm-l6-v2` 라이브러리에 포함

**ChromaDB와 TimescaleDB 역할 분리**
| DB | 저장 내용 | 용도 |
|---|---|---|
| TimescaleDB | 뉴스 원문, 감성, URL 등 전체 | 일반 조회 |
| ChromaDB | 벡터 + 메타데이터(일부 필드) | 유사도 검색 |

### NewsVectorService.java

// 저장: 텍스트 → 벡터 변환 → ChromaDB 저장
public void storeNewsEmbedding(Long newsId, String headline, String summary, String sentiment, String url) {
    String text = headline + " " + summary;
    Metadata metadata = new Metadata();
    metadata.put("newsId", ...), metadata.put("sentiment", ...), metadata.put("url", ...);

    Embedding embedding = embeddingModel.embed(TextSegment.from(text, metadata)).content();
    embeddingStore.add(embedding, segment);
}

// 검색: 검색어 벡터 변환 → 유사도 검색
public List<EmbeddingMatch<TextSegment>> searchSimilarNews(String query, int maxResults) {
    Embedding queryEmbedding = embeddingModel.embed(query).content();
    return embeddingStore.findRelevant(queryEmbedding, maxResults);
}

// 마이그레이션: TimescaleDB → ChromaDB 일괄 이동
public int migrateExistingNews() {

}

- `AuthModal.jsx`: shadcn Dialog + Input + Label + Button
- `PricePage.jsx`: shadcn Card + recharts AreaChart (카드 클릭 시 차트 토글)
- `NewsPage.jsx`: shadcn Card + Badge, 감성 필터 버튼
- `SearchPage.jsx`: shadcn Input + Button + Card + Badge

### 차트 동작 방식
1. PricePage에서 BTC 또는 ETH 카드 클릭
2. `GET /api/trades/history?symbol=BINANCE:BTCUSDT&limit=60` 호출
3. 최근 60건 데이터를 시간순으로 정렬하여 recharts AreaChart로 표시
4. 같은 카드 다시 클릭 또는 "닫기" 버튼으로 차트 숨김

### 주요 설정
- shadcn 다크 테마 (oklch 컬러 시스템)
- BTC 차트 색상: `#f7931a` (비트코인 오렌지)
- ETH 차트 색상: `#627eea` (이더리움 퍼플)
- recharts AreaChart: `dot={false}`, `type="monotone"`, 그라디언트 fill
